## 短期目标 (v3.9.x)

### 1. 流式输出支持 ⭐⭐⭐

**优先级**: P0 (最高)  
**预计工作量**: 2-3天  
**目标版本**: v3.9.0

**问题描述**:
- 当前 API 调用是阻塞式的，用户需要等待完整回复
- 大模型回复较长时，等待时间过长（5-10秒）
- 用户体验较差

**解决方案**:
```javascript
// 支持 streaming response
async function callAPIStreaming(message, onChunk) {
    // 使用 ReadableStream 实时接收数据
    const response = await fetch(url, { ... });
    const reader = response.body.getReader();
    while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        onChunk(decode(value)); // 实时显示
    }
}
```

**实施步骤**:
- [ ] 修改 api.js 支持 SSE (Server-Sent Events)
- [ ] 修改 chat.js 处理流式数据
- [ ] 修改 ui.js 支持增量更新消息
- [ ] 添加"正在输入..."动画
- [ ] 测试不同模型的流式支持

**验收标准**:
- ✅ 用户发送消息后 1 秒内看到首字
- ✅ 支持 OpenRouter 免费模型的流式输出
- ✅ 停止按钮可以中断流式输出
- ✅ 流式输出完成后触发代码执行

**收益**:
- 用户体验提升 80%
- 感知延迟降低
- 支持更长的回复
