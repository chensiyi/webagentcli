## 🎯 当前重点：多语言界面支持 (v4.0.0)

### 4. 多语言界面支持 ⭐⭐⭐

**优先级**: P1 (当前重点)  
**预计工作量**: 2-3天  
**目标版本**: v4.0.0  
**状态**: 待开发

**问题描述**:
- 仅支持中文界面，无法服务国际化用户
- UI 文本硬编码在代码中，难以维护
- 缺乏语言切换机制

**解决方案**:
```javascript
// i18n 系统架构
const I18N = {
    currentLang: 'zh',  // 当前语言
    supportedLangs: ['zh', 'en', 'ja'],  // 支持的语言
    translations: {
        zh: { settings: '设置', send: '发送', ... },
        en: { settings: 'Settings', send: 'Send', ... },
        ja: { settings: '設定', send: '送信', ... }
    },
    t(key) { return this.translations[this.currentLang][key]; }
};
```

**实施步骤**:
- [ ] **步骤 1**: 创建 `core/i18n.js` 模块
  - 语言配置管理
  - 翻译函数 t(key, params)
  - 语言切换函数 setLanguage(lang)
  - 自动检测浏览器语言
  
- [ ] **步骤 2**: 提取 UI 文本（按模块）
  - `ui-templates.js` - 主界面、设置对话框
  - `ui.js` - 按钮文本、提示消息
  - `chat.js` - 欢迎消息、帮助信息
  - `models.js` - 模型名称、分组标题
  
- [ ] **步骤 3**: 实现三种语言翻译
  - ✅ 中文 (zh) - 默认语言
  - [ ] 英文 (en) - 国际化
  - [ ] 日文 (ja) - 亚洲用户
  
- [ ] **步骤 4**: 添加语言切换功能
  - 设置面板添加语言选择下拉框
  - 切换后即时更新界面（无需刷新）
  - 语言配置持久化（LocalStorage）
  
- [ ] **步骤 5**: 测试和验证
  - 测试所有界面的语言切换
  - 验证翻译完整性
  - 检查文本溢出问题
  - 测试语言配置保存/加载

**验收标准**:
- ✅ 所有界面文本可翻译（UIStyles、UITemplates、UIManager）
- ✅ 支持中文、英文、日文 3 种语言
- ✅ 切换语言即时生效（无需刷新页面）
- ✅ 语言配置持久化（按域名隔离）
- ✅ 自动检测浏览器语言（首次使用）
- ✅ 文本溢出自适应（不同语言长度差异）
- ✅ 快捷键提示同步翻译

**技术细节**:
```javascript
// 文本键命名规范
const UI_KEYS = {
    // 通用按钮
    'ui.send': '发送',
    'ui.stop': '停止',
    'ui.clear': '清空',
    'ui.settings': '设置',
    
    // 设置对话框
    'settings.title': 'AI Agent 设置',
    'settings.apiKey': 'API Key',
    'settings.model': '模型',
    
    // 消息提示
    'msg.welcome': '欢迎使用 Free Web AI Agent',
    'msg.error.apiKey': '请设置 API Key',
    
    // 代码执行
    'code.confirm': '确认执行此代码？',
    'code.running': '执行中...',
    'code.success': '✅ 执行成功',
};
```

**文件修改计划**:
1. **新建**: `src/core/i18n.js` - 国际化核心模块
2. **修改**: `src/ui-templates.js` - 提取模板文本
3. **修改**: `src/ui.js` - 使用 t() 函数替换硬编码
4. **修改**: `src/chat.js` - 提取消息文本
5. **修改**: `src/main.js` - 初始化 i18n 模块
6. **修改**: `build.js` - 添加 i18n.js 到构建列表

**收益**:
- 国际化，服务全球用户
- 代码可维护性提升（文本集中管理）
- 为未来扩展更多语言打下基础
- 提升产品专业度

**风险与注意事项**:
- ⚠️ 不同语言文本长度差异（日文通常比英文短）
- ⚠️ 需要维护 3 份翻译文件
- ⚠️ 用户添加的自定义文本不翻译（如代码块、用户消息）
- ⚠️ 模型名称、API 错误消息可能保持英文
